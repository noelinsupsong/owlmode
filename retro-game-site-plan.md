## 1. 프로젝트 개요

### 1.1 서비스 소개

무료로 즐길 수 있는 레트로 스타일 웹 게임 플랫폼. 별도 설치 없이 브라우저에서 바로 플레이할 수 있으며, 구글 AdSense를 통해 비침습적인 광고 수익을 창출한다.

### 1.2 핵심 목표

- 고품질 레트로 게임을 무료로 제공
- 깔끔하고 비침습적인 광고 배치로 사용자 경험 유지
- 오픈소스 기반으로 법적 리스크 없이 운영
- 빠른 로딩과 모바일 대응으로 넓은 접근성 확보

### 1.3 타겟 사용자

| 항목 | 내용 |
|------|------|
| 주 타겟 | 10대 ~ 30대 |
| 특성 | 레트로 감성, 캐주얼 게이머, 잠깐의 여가를 즐기는 유저 |
| 접속 환경 | PC 우선, 모바일 지원 |

## 2. 기술 스택

### 2.1 프론트엔드

| 항목 | 기술 |
|------|------|
| 프레임워크 | **Next.js 14 (App Router)** |
| 언어 | TypeScript |
| 스타일링 | Tailwind CSS |
| 게임 렌더링 | HTML5 Canvas / 각 게임별 순수 JS |
| 국제화(i18n) | next-intl (한국어 / 영어) |

> **Next.js를 선택한 이유:**
> - SEO에 유리한 SSR/SSG 지원 → 구글 검색 유입 극대화
> - AdSense와 호환성 우수
> - 페이지 라우팅이 깔끔하여 게임별 URL 분리 용이

### 2.2 배포 및 인프라

| 항목 | 기술 |
|------|------|
| 호스팅 | Vercel (무료 플랜 시작) |
| 도메인 | 별도 구매 권장 |
| CDN | Vercel Edge Network 기본 포함 |

## 3. 게임 목록 (인기순)

오픈소스 라이선스 기준: **MIT / Apache 2.0 / Public Domain** 만 사용.  
직접 구현(from-scratch) 게임은 클래식 게임 규칙을 참고하되 코드는 100% 자체 작성.

### 3.1 1차 출시 게임 (8종)

| 순위 | 게임명 | 장르 | 구현 방식 | 참고 라이선스 |
|------|--------|------|-----------|---------------|
| 1 | **2048** | 퍼즐 | 자체 구현 (규칙 Public Domain) | MIT 참고 |
| 2 | **테트리스 클론** | 블록 쌓기 | 자체 구현 (Tetris 규칙 공개) | 자체 작성 |
| 3 | **뱀 게임 (Snake)** | 아케이드 | 자체 구현 | 자체 작성 |
| 4 | **지뢰찾기** | 퍼즐 | 자체 구현 | 자체 작성 |
| 5 | **Flappy Bird 클론** | 아케이드 | 자체 구현 | 자체 작성 |
| 6 | **Pac-Man 클론** | 아케이드 | 자체 구현 (AI 경로 포함) | 자체 작성 |
| 7 | **Breakout (벽돌 깨기)** | 아케이드 | 자체 구현 | 자체 작성 |
| 8 | **Sudoku** | 퍼즐 | 자체 구현 + 문제 생성 알고리즘 | 자체 작성 |

> ⚠️ **법적 안전 원칙:**
> - 게임의 **코드, 에셋(스프라이트·사운드)** 은 100% 자체 제작하거나 CC0/Public Domain 소스만 사용
> - 기존 오픈소스를 fork하여 사용할 경우 MIT/Apache 2.0 라이선스 표기 의무 준수
> - 상표 이슈가 있는 원작 이름은 사용하지 않고 "클론" 또는 자체 이름 부여
> - Tetris는 The Tetris Company가 상표 보유 → "Block Stack", "TileDown" 등 자체 명칭 사용

- Pong 클론 (1P vs CPU)

## 6. 광고 정책 (Google AdSense)

### 6.1 광고 배치 원칙

- **팝업 광고 없음** (절대 금지)
- **전면 광고 없음** (게임 진행 방해 금지)
- **자동재생 영상 광고 없음**

### 6.2 광고 위치 (최대 2곳)

| 위치 | 형식 | 페이지 |
|------|------|--------|
| 메인 헤더 하단 | 가로 배너 (728x90 또는 반응형) | 랜딩 페이지 |
| 게임 사이드바 | 직사각형 (300x250) 또는 세로형 | 게임 페이지 |

> 모바일에서는 게임 하단 고정 배너 1개로 통합

### 6.3 AdSense 승인 요건 대응

- 고유 콘텐츠 (자체 제작 게임) 확보
- Privacy Policy 페이지 필수 추가
- 최소 페이지 수 확보 (게임 8개 + 메인 + about)
- 로딩 속도 최적화 (Core Web Vitals 충족)
